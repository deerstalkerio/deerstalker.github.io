Architecture

This document outlines the high level system architecture for Deerstalker.io. This document outlines the internal components for how Deerstalker which allow you to monitor your applications.

System Context

Block Diagram

This section defines how deerstalker ecosystem breaks up into individual components.

Deerstalker API

The Deerstalker API exposes endpoints within which allow the deerstalker site, to present the application metrics.

Identity

This provides very basic user management, providing authentication for the presentation layer as well as the API. You can manage these users through the deerstalker command line client or using the admin panel.

Metrics

These generically represent all the metrics which have been submitted by your applications.

Plugins

Deerstalker Ingest

The Deerstalker Ingest allows for data to be pushed at any frequency and thus then pushed into the backend engesr flow.
